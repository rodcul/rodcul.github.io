Feature 2 — warm start

Setup, the domain model, and the safety net are already paid. You drop straight into the loop:

/clear first — new feature, fresh window. CONTEXT.md re-grounds the agent instantly.

  1. Align/grill-with-docs (not bare /grill-me)Re-grills this feature and updates CONTEXT.md — prefer it so the decisions persist and you can clear right after. A bare /grill-me leaves decisions in the chat only.
    CONTEXT.md (updated)clear-safe
  2. Spec→Slice/to-prd → /to-issues → /triagePRD + slice issuesclear-safe
  3. Build→Fix"Implement test-first." tdd and diagnosing-bugs auto-fire.tests + source, committedclear between issues
Refresh context as often as you can The artifacts are the memory, so /clear after any step that wrote a file or a tracker issue — the next step rehydrates from it. Clear between every issue, every stage, every feature; short windows keep the agent sharp. The one rule: never clear after a step whose output lives only in the chat (a bare /grill-me) — persist it first, or use /handoff. Cold-start cost (setup + domain model + safety net) is paid once; every later feature is just grill-with-docs → prd → issues → tdd. Lost? /ask-matt routes you.
