The Safety Net & Glue

The loop runs fast and clean only because something underneath it refuses to let it regress — and a little glue keeps you moving between the stages.

You have walked the whole loop: align, spec, slice, build, fix, tend. This final lesson is the part that is always on but rarely seen — the safety net that stops the agent from quietly breaking things, plus the small glue skills that connect the stages and carry work between sessions. — mattpocock/skills

The safety net — so the loop cannot silently regress

SkillWhat it guards
git-guardrails-claude-codeBlocks dangerous git commands before they execute — no accidental force-push or history rewrite from an over-eager agent.
setup-pre-commitWires Husky + lint-staged + Prettier + typecheck + tests as commit gates, so broken or unformatted code cannot land.

Together these mean the discipline is not a thing you have to remember. The agent literally cannot commit code that fails the gates, and cannot run the git commands that lose work. The floor holds even on a tired Friday.

The glue — keep moving between skills and sessions

SkillKindUse it to…
ask-mattUSERRoute yourself — when unsure which skill fits the moment, it points you at the right one.
handoffUSERCompact the current conversation into a clean handoff another agent can pick up cold.
prototypeUSERBuild a throwaway prototype — a terminal app or UI variations — to learn fast before committing to a design.
Why glue matters The loop is not a straight line you walk once. You jump back to grilling mid-build, hand work to a fresh agent when context fills, and throw away a prototype to learn what to actually spec. The glue skills are what make those moves cheap instead of disruptive.
