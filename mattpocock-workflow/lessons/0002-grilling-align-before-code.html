Grilling: Align Before Code

Flip the interview around — let the agent interrogate you until every ambiguous decision is dead.

Lesson 1 ended on a principle: alignment before code. An agent that misunderstands you builds the wrong thing flawlessly, fast. Grilling is the stage that makes that impossible. Instead of you writing a careful prompt and hoping, the agent interviews you — relentlessly — surfacing every unstated assumption and unresolved branch before a single line is written. — mattpocock/skills

Three skills, one loop

Grilling is built from two skills you invoke and one the model runs underneath them.

SkillKindWhat it does
grill-me USER A relentless interview until every decision branch is resolved. Pure clarity — it leaves no file behind, just aligned intent.
grill-with-docs USER The same interview, but it also builds the domain model and writes it into CONTEXT.md.
grilling MODEL The reusable interview loop underneath both of the above. You rarely call it directly.

The split is the whole point. Reach for grill-me when you just need to think a problem through out loud and get unstuck. Reach for grill-with-docs when the clarity you produce should persist as shared vocabulary the next session can read. — mattpocock/skills

The mindset shift You are not the one being helpful here — the agent is. A good grilling feels uncomfortable: it keeps asking "what about…?" until you have no vague answers left. That discomfort is the alignment.

Check yourself

Recall before you reveal — the effort is what makes it stick.

Your win: grill out a real CONTEXT.md

Tangible win By the end of this step the anchor repo has a CONTEXT.md capturing its real domain language — product site, SEO agent, and the terms your future sessions will lean on to stay terse.

Pick a genuine upcoming change — say, a new capability for the example-service — and let the agent grill you on it in the anchor repo:

# in ~/code/example-app
/grill-with-docs    # answer honestly; resist giving vague answers

Let it push. When it asks something you have not decided, decide it out loud — that is the work. When it finishes, read the CONTEXT.md it produced and check the vocabulary matches how you actually talk about the system. Fix anything that reads like the agent guessing. Skill: grill-with-docs in mattpocock/skills.

Watch for If the grilling feels too shallow, that is usually because you gave a confident-but-vague answer it accepted. Tell it to dig harder on that branch — or tell your teacher and we will sharpen the prompt.
🧹 Clear here for best performance You just wrote CONTEXT.md to disk, so /clear now. A short, focused context is where an LLM does its sharpest work — less to attend to means tighter reasoning, less drift onto stale chatter, and lower cost and latency. The next stage (/to-prd) rehydrates straight from CONTEXT.md + the codebase. One caveat: a bare /grill-me writes no file — persist its decisions first (or use /grill-with-docs) before you clear.
